1819で描画処理を変更しました
この変更により、フェードのような画面効果の再生中に画面がちらつく場合があります。
これは今まで表面化しなかったERBスクリプトのバグの顕在化です。
この問題は1818以前のEmueraでも「フレーム毎秒」を非常に大きな値にすることにより再現できます。

（１）エンドユーザー様へ
Emueraのコンフィグで「表示」タブの「フレーム毎秒」を1に設定してください
おそらくちらつかなくなります。


（２）ERB開発者様へ
画面のちらつきはCLEARLINEを行った瞬間の状態が描画されてしまうことが原因です。
フェードのような、CLEARLINEを使用した画面の書き換えを頻繁に行う処理をした場合、REDRAW命令を利用してください。
	REDRAW 0
	CLEARLINE x
	PRINTL hoge
	REDRAW 1
	TWAIT 100,1
REDRAW 0 により描画処理が抑制され、次にREDRAW 1を行うかユーザーの入力待ち状態になるまでの間画面描画が行われなくなります
結果としてCLEARLINE後、PRINT前に描画が行われなくなり、画面のちらつきを防ぐことができます。

なお、1819でもこの問題が発生しない場合であっても、潜在的なバグであることは変わりません。
開発環境において画面がちらつかなかったとしてもユーザーの環境や「フレーム毎秒」の設定次第でちらつく可能性があります。
REDRAW命令を活用してください。
